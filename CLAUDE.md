# CLAUDE.md - Expense Tracker

## Project Overview
Personal expense management app built with Laravel 12. Features: expense tracking, budgets, recurring expenses, savings goals, AI analysis (Gemini), reports (PDF/Excel), Google OAuth, multi-language (en/vi).

## Tech Stack
- **Backend**: PHP 8.2+, Laravel 12
- **Frontend**: Blade templates, Tailwind CSS v4, Chart.js, Vite v7
- **Database**: MySQL (SQLite for tests)
- **External Services**: Google OAuth (Socialite), Google Gemini API, Mailtrap SMTP
- **Libraries**: dompdf (PDF), Maatwebsite/Excel (Excel export)

## Common Commands
```bash
# Dev server
php artisan serve
yarn dev              # Vite dev server

# Build
yarn build

# Database
php artisan migrate
php artisan migrate:fresh --seed

# Tests
php artisan test

# Queue
php artisan queue:work

# Scheduled commands
php artisan expenses:create-recurring    # --dry-run supported
php artisan budgets:check-thresholds

# Cache
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## Project Structure
```
app/
  Http/Controllers/     # 21 controllers (Auth, Admin, Core, Features)
  Http/Middleware/       # ActiveUser, Admin, Onboarding, SetLocale
  Models/               # User, Expense, Category, Budget, RecurringExpense, SavingsGoal, etc.
  Services/             # BudgetAlertService, GeminiService
  Jobs/                 # SendBudgetAlertEmailJob
  Mail/                 # BudgetAlertMail
  Console/Commands/     # CreateRecurringExpenses, CheckBudgetThresholds
routes/web.php          # All routes (no API routes)
resources/views/        # Blade templates
lang/{en,vi}/           # Translation files (messages.php, validation.php)
database/migrations/    # 17 migrations
config/                 # App, services (Google, Gemini), onboarding, etc.
```

## Code Conventions
- **Validation**: Inline in controllers via `$request->validate()` (no Form Request classes)
- **Authorization**: Manual `Auth::id()` checks with `abort(403)`
- **Database**: Eloquent ORM only, eager loading with `with()`, mass assignment via `$fillable`
- **Services**: Constructor DI (BudgetAlertService), static factory (GeminiService::fromConfig())
- **i18n**: `__('messages.key')` for all user-facing strings
- **Views**: Blade components, view composers for shared data (notifications)
- **Money**: Decimal(12,2) in database

## Middleware Chain (protected routes)
`auth` -> `active` -> `onboarding` -> controller
Admin routes add `admin` middleware.

## Key Patterns
- No repositories - controllers use models directly
- Service layer for complex business logic only
- Budget alerts at 50%, 80%, 100% thresholds via queued jobs
- Recurring expenses generated by artisan command (cron)
- Notifications stored in DB, shared to views via AppServiceProvider composer

## Testing
- PHPUnit 11, SQLite in-memory for tests
- Test suites: Unit, Feature
- Config: `phpunit.xml`

## Important Notes
- NEVER read or output `.env` file
- All user-facing text must use translation keys from `lang/` files
- Queue driver is `database` - run `queue:work` for async jobs
- Gemini model default: `gemini-2.5-flash` (configured in `config/services.php`)
